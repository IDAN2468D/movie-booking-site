# 🤖 Agent Skill Definition: hyper_realistic_web_scratch_card_generator

## 📋 Overview
Generates hyper-realistic, hardware-accelerated pure web scratch card modules for Next.js 15. It combines opaque physical material rendering, rough-edge pixel erasure masking to simulate physical debris, procedural audio synthesis, and a zero-overlap architectural ticket layout.

## 🧱 Pure Web Tech Stack
- **Framework:** Next.js 15 (App Router) & TypeScript.
- **Masking Engine:** HTML5 Canvas API (`2d` context) utilizing `globalCompositeOperation = 'destination-out'` with customized rough-edge brush properties (`shadowBlur`, brush alpha scattering).
- **Styling & Theme:** Tailwind CSS 4.0 (Opaque space-black materials, cybernetic gradients, gold accents, absolute layouts).
- **Audio Engine:** Web Audio API (`AudioContext`, `BiquadFilterNode`, `AudioBufferSourceNode`) generating procedural friction noise mapped to cursor tracking speed.

## 📐 Material & Performance Rules
1. **100% Opacity Mask Rule:** The top cover mask MUST be entirely opaque using `#0D0F14` to `#151821` to eliminate underlying text bleeding or visual spoilers. It must render a subtle, low-opacity cyber-monogram neural grid pattern over the surface.
2. **Rough-Edge Brush Physics:** The canvas eraser brush must avoid perfectly clean vector lines. It must integrate a soft, feathered, or jagged contour to realistically mimic physical latex scratching.
3. **Atomic File Constraint:** Every component and hook must be fully modular and stay strictly under 200 lines of code.
